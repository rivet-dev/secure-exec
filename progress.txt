- **Learnings for future iterations:**
  - crossterm 0.28.1 (used by ratatui 0.29) has no WASI support — requires vendor patch adding `#[cfg(target_os = "wasi")]` modules for terminal, cursor, event source, tty, filter, and InternalEvent variants
  - ratatui re-exports crossterm as `ratatui::crossterm` — don't add crossterm as a separate dep, use the re-export

  - EventFilter in filter.rs has platform-specific eval implementations — must add WASI variant
  - KeyModifiers::CONTROL display in event.rs has `cfg(unix)` and `cfg(windows)` but no fallback — must add WASI variant
  - CrosstermBackend's `writer_mut()` is private in ratatui 0.29 — use `io::stdout()` directly for LeaveAlternateScreen
  - WASI terminal raw mode is tracked as AtomicBool in-process (host PTY handles actual discipline)
  - WASI terminal size reads COLUMNS/LINES env vars, defaults to 80x24
  - WASI event source reads raw bytes from stdin and uses crossterm's parse_event for ANSI parsing
