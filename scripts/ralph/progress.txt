## Codebase Patterns
- Run upstream Node bootstrap probes in isolated child host Node processes with `--expose-internals`; `internal/bootstrap/node` can abort on native asserts and kill the current test worker if run inline.
- For bootstrap preflight work, read builtin source from `process.binding('natives')` and defer pinned vendored asset sync to the dedicated upstream-asset story instead of partially vendoring bootstrap files early.
- WasmVM socket tests that create client sockets from the test harness must register a real kernel-owned PID first; fake PID literals like `999` are rejected by `SocketTable`.
- When WasmVM `host_net` imports change, rebuild `@secure-exec/wasmvm` before validating probes; runtime execution uses `packages/wasmvm/dist/kernel-worker.js`, not just `src/kernel-worker.ts`.
- Upstream Node asset sync must export from a pinned git ref and validate with `pnpm check:node-upstream-assets`; do not copy builtin files from a dirty checkout or from upstream `main` when the story is pinned to a release tag.
- `pnpm check:node-upstream-assets` only needs a git checkout that contains the pinned commit; a temporary `git init` plus single-commit fetch is enough for real-release validation.
