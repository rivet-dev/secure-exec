## Codebase Patterns
- Integration tests use `createIntegrationKernel({ runtimes: ['wasmvm', 'node'] })` from `packages/secure-exec/tests/kernel/helpers.ts`
- All kernel integration tests are guarded with `skipUnlessWasmBuilt()` and use `describe.skipIf(skipReason)`
- `kernel.exec(command)` returns `{ exitCode, stdout, stderr }` — routes through shell
- `kernel.spawn(cmd, args, { onStdout, onStderr })` for raw process spawning with output callbacks
- TerminalHarness wraps `kernel.openShell()` + headless xterm for PTY testing: `waitFor(text, occurrence, timeout)`, `type(input)`, `screenshotTrimmed()`
- PROMPT constant for brush-shell is `'sh-0.4$ '`
- Use `findOutputLine(screen, expected)` to distinguish real output from command echo on terminal
- Each test creates a fresh kernel via `afterEach(async () => { await ctx?.dispose(); })`
- ExecOptions supports `stdin`, `env`, `cwd`, `timeout`, `onStdout`, `onStderr`
- Test file naming: use `behavior`, `suite`, `integration`, `parity`, `policy` — never `contract`
- PTY session leader SIGINT: when `sessionLeaderPgid` is set, ^C echoes "^C\r\n" and injects "\n" instead of killing the session leader via driver.kill()
- `killGroupExcludeLeaders(pgid, signal)` on ProcessTable skips processes where pid === sid (session leaders)
- Permission enforcement in kernel-worker.ts: all write syscalls gate with `isWriteBlocked()`, all process/resource syscalls gate with `isSpawnBlocked()` — audit both when adding new syscall handlers

## 2026-03-19 - US-036

- Root cause: PTY correctly detects SIGINT and routes it to foreground pgid, but WasmVM driver.kill() always calls worker.terminate() which destroys the shell
- Fix: PTY-level session leader SIGINT interception — when foreground pgid matches session leader's pgid, PTY echoes "^C\r\n", clears canonical buffer, and injects newline to trigger fresh prompt instead of killing the WASM worker

  - `packages/kernel/src/process-table.ts` — killGroupExcludeLeaders method

  - `packages/secure-exec/tests/kernel/ctrl-c-shell-behavior.test.ts` (new) — 4 integration tests with real WasmVM
- **Learnings for future iterations:**
  - WasmVM driver.kill() is destructive (worker.terminate()) — no graceful signal delivery to WASM processes
  - The WASI layer does NOT expose tcsetattr/tcgetattr/tcsetpgrp — brush-shell runs in default canonical+echo+isig mode
  - Session leader (pid === sid) identification works because openShell() sets shell's pgid = pid, and sid defaults to pid for root processes
  - When brush-shell spawns children, they inherit the shell's pgid (no setpgid WASI import), so foreground pgid always equals session leader's pgid
  - The `deliverInput()` bypass delivers raw bytes to slave input without going through canonical processing — used to inject newline for prompt refresh

- Gated fd_dup with isSpawnBlocked() — prevents resource exhaustion from restricted tiers
- Added 4 integration tests to driver.test.ts permission tier enforcement section:
  - read-only command cannot write via pwrite path
  - read-only command calling proc_kill is blocked
  - isolated command cannot create pipes (fd_pipe blocked)

  - `packages/runtime/wasmvm/test/driver.test.ts` — added 4 permission enforcement tests
- **Learnings for future iterations:**
