## Codebase Patterns
- MockRuntimeDriver `kill()` now calls `proc.onExit?.(code)` — always wire onExit in custom test stubs that need FD cleanup
- Tests doing async FD operations (fdRead, fdWrite) must use `neverExit: true` to prevent process from exiting mid-test; kill explicitly at end
- `fdTableManager.remove(pid)` closes all FDs and removes the table; pipe ends need separate `pipeManager.close()` calls for descriptions whose refCount drops to 0
- Kernel converts pipe EOF (null) to empty `Uint8Array(0)` in fdRead — test for `.length === 0` not `null`
- Bridge http.request() only supports `(options, callback)` and `(url, callback)` — NOT the 3-arg `(url, options, callback)` form
- Bridge HTTP server: Express works as `http.createServer(app)` callback; Fastify v5 routing does NOT work with bridge's ServerIncomingMessage
- Project-matrix sandbox: requires `createDefaultNetworkAdapter()` + `allowAllNetwork` permissions for HTTP fixtures
