## Codebase Patterns
- Active module-load benchmark artifacts live in `packages/secure-exec/benchmarks/results/module-load/`; refresh them with the full `pnpm --filter secure-exec bench:module-load` run after each performance story.
- Module-load benchmark `summary.md` / `comparison.md` now split `_loadPolyfill` into `real polyfill-body loads` and `__bd:* bridge-dispatch wrappers`; copy those fields directly for progress updates instead of using ad hoc probes.
- `comparison.md` frame-byte deltas average all benchmark iterations; when a story changes cold-vs-warm `Execute` payloads, inspect the per-scenario `ipc.ndjson` `ipc_frame` entries directly. Warm bridge snapshot-ref hits show `bridgeCodeBytes = 0` with non-zero `bridgeCodeRefBytes`.
- Snapshot-preloaded builtin polyfill bundles must use the same normalized module list in both `createNodeV8Runtime()` and `createNodeRuntimeDriverFactory()`; otherwise the warmup bridge source and per-execute bridge source diverge and fresh-session `Execute` misses the warmed snapshot path.
- Module-load scenario reruns load `@secure-exec/v8` from `packages/v8/dist`; after changing `packages/v8/src/*` behavior that the benchmark exercises, rebuild with `pnpm --filter @secure-exec/v8 build` before trusting fresh artifacts.
- When deriving module-load baselines from committed `ipc.ndjson` via `git show`, set a large `maxBuffer`; Pi logs exceed Node's default sync buffer.
- Module-load scenario reruns must start from a truncated `ipc.ndjson`; the V8 observability sink appends by default, so stale logs will corrupt per-scenario summaries and `comparison.md`.
- Shared `createNodeV8Runtime()` reuse now depends on `session.destroy()` fully draining the runtime-side teardown path; if a workload generates lots of timer or stream events, use an ordered barrier like `DestroySession` -> `Ping/Pong` instead of treating session destroy as fire-and-forget.
- Headless CLI workloads inside standalone `NodeRuntime.exec()` should pass `stdin: ""`; for Pi's direct `dist/main.js` print-mode path with an explicit `--print` message, set `process.stdin.isTTY = true` inside the sandbox harness or Pi stalls in its piped-stdin probe before provider traffic.
- Promise-heavy standalone `NodeRuntime.exec()` benchmark scripts need an explicit live handle (for example a short `setInterval()` cleared in `finally`) until the final stdout JSON is emitted, or exec-mode runs can return `code: 0` before long async library work like `pdfDoc.save()` or `zip.generateAsync()` produces output.
- Sandbox `setImmediate()` must use a queue-drain path, not `setTimeout(0)`; JSZip/readable-stream worker pipelines rely on deferred-immediate semantics, and zero-delay timers can turn that into a spin/hang.
- For Pi startup and end-to-end perf stories, record warm wall time together with bridge calls/iteration and dominant IPC counters (`_loadPolyfill`, BridgeResponse bytes, and any new network callsites) instead of relying on a single aggregate number.
- Module-load benchmark progress reporting now comes from generated `packages/secure-exec/benchmarks/results/module-load/comparison.md` plus per-scenario `summary.json`; fixed session overhead is `sample.wallMs - ipc_execute.finish.durationMs` for the matching session in `ipc.ndjson`.
- Module-load phase attribution now comes from `CreateSession -> InjectGlobals -> Execute -> ExecutionResult -> DestroySession` timestamps in per-scenario `ipc.ndjson`, and raw authenticated IPC RTT lives in top-level `transport-rtt.json` / `transport-rtt.md`.
- Snapshot-hoisted `postRestoreScript` work in `packages/nodejs/src/execution-driver.ts` must keep `require`/console/bootstrap setup ahead of timing-freeze hardening, while custom-global exposure stays after timing mitigation; otherwise freeze mode locks `SharedArrayBuffer` before `require-setup` re-exports it and warm execs fail with `Cannot redefine property: SharedArrayBuffer`.
- Sync module-resolution referrer normalization must only trim real module file extensions; dotted package directories like `yaml-1.1/` are valid directories, and treating any dotted suffix as a file forces package-internal relative requires down the async `_resolveModule` fallback.
