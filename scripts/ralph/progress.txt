## Codebase Patterns

- All sandbox tests use `createTestNodeRuntime()` or `NodeRuntime` directly, then `proc.exec()` / `proc.run()` to execute code inside the VM
- Look at `tests/test-suite/node.test.ts` for the canonical pattern of creating a runtime and running sandbox code
- Mock LLM server at `tests/cli-tools/mock-llm-server.ts` is fine running on host ‚Äî sandbox code reaches it via network bridge
- Pi has `createAgentSession()` programmatic API in `@mariozechner/pi-coding-agent` (dist/index.js) ‚Äî designed for in-process headless use
- Claude Code SDK always spawns cli.js as subprocess (ProcessTransport class) ‚Äî no in-process execution path
- OpenCode npm package ships only a compiled Bun binary (ELF) ‚Äî no JS source to load in VM
- Interactive tests should use `kernel.openShell()` with `@xterm/headless`, not host PTY via `script -qefc`

- ModuleAccess overlay with pnpm: use WORKSPACE ROOT as `moduleAccess: { cwd }` to expose transitive deps via `.pnpm/node_modules/` hoisted path
- pnpm transitive deps are in workspace root `node_modules/.pnpm/node_modules/`, not package-level
- `wrapFileSystem` in permissions.ts strips custom methods ‚Äî extract `toHostPath` before wrapping via `sandboxToHostPath` in execution-driver
- V8 isolated-vm does NOT set import.meta.url ‚Äî ESM compiler polyfills it with `file://` URL
- V8 isolated-vm throws on conflicting star exports (Node.js makes them ambiguous) ‚Äî `deconflictStarExports` in esm-compiler.ts handles this
- CJS-in-ESM wrapper must use `let exports` (not `const`) ‚Äî CJS code commonly reassigns exports binding
- network.ts is the authoritative net module implementation (TLS, _readableState, Buffer proto) ‚Äî net.ts was removed
- For module resolution inside the sandbox, set `filePath: '/root/entry.js'` and `cwd: '/root'` in `exec()` options ‚Äî the overlay mounts at `/root/node_modules`
- `NodeFileSystem` as base filesystem for `createNodeDriver` provides real host filesystem access for file I/O
- `createDefaultNetworkAdapter({ initialExemptPorts })` bypasses SSRF checks for specific localhost ports
- `createHostCommandExecutor()` wraps `node:child_process.spawn` as a `CommandExecutor` for sandbox bash tool tests
- TCP socket bridge: guest calls `_netSocketConnectRaw.applySync()` ‚Üí host creates real `net.Socket` ‚Üí events dispatched back via `_netSocketDispatch` applySync callback
- `wrapNetworkAdapter` in `permissions.ts` must forward ALL adapter methods ‚Äî new methods added to NetworkAdapter interface need corresponding forwarding in the wrapper
- pg library uses Web Crypto `subtle.deriveBits` for SCRAM-SHA-256 auth ‚Äî implemented in sandbox crypto.subtle bridge (guest: SandboxSubtle.deriveBits in require-setup.ts, host: deriveBits case in bridge-setup.ts using pbkdf2Sync)
