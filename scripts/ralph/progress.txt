## Codebase Patterns
- Active module-load benchmark artifacts live in `packages/secure-exec/benchmarks/results/module-load/`; refresh them with the full `pnpm --filter secure-exec bench:module-load` run after each performance story.
- The top-level module-load driver should orchestrate scenario stages in isolated child processes; only the `samples` stage needs to own the committed `ipc.ndjson` / `metrics.prom`, while the benchmark-mode stages can return partial benchmark-mode JSON that the parent merges into the final `result.json`.

- Package type checks in this repo usually live behind `check-types` or `check-types:benchmarks`, not a generic `typecheck` script.
- Module-load benchmark `summary.md` / `comparison.md` now split `_loadPolyfill` into `real polyfill-body loads` and `__bd:* bridge-dispatch wrappers`; copy those fields directly for progress updates instead of using ad hoc probes.
- Module-load `_loadPolyfill` hotspot summaries must keep an explicit unattributed remainder bucket when older `ipc.ndjson` logs lack `bridgeTarget`; otherwise target tables silently undercount legacy traffic.
- Hot `__bd:*` dispatch wrappers should use the dedicated `_bridgeDispatch` bridge global; keep `_loadPolyfill("__bd:...")` parsing only as a compatibility fallback for older snapshots and logs.
- Cross-session V8 BridgeResponse caching can now store any V8-serializable value, not just strings; for `_bridgeDispatch` `_loadFile` / `_loadFileSync` source envelopes, key cache reuse on the dispatch method, request args, and serialized payload hash because transformed module bodies are path-sensitive.
- When adding a host bridge global, update the shared bridge-contract exports, `packages/core/src/shared/global-exposure.ts`, runtime-global typings, and the native V8 sync/async bridge registry together; the bridge-registry policy tests treat drift as a real failure.
- `comparison.md` frame-byte deltas average all benchmark iterations; when a story changes cold-vs-warm `Execute` payloads, inspect the per-scenario `ipc.ndjson` `ipc_frame` entries directly. Warm bridge snapshot-ref hits show `bridgeCodeBytes = 0` with non-zero `bridgeCodeRefBytes`.
- Snapshot-preloaded builtin polyfill bundles must use the same normalized module list in both `createNodeV8Runtime()` and `createNodeRuntimeDriverFactory()`; otherwise the warmup bridge source and per-execute bridge source diverge and fresh-session `Execute` misses the warmed snapshot path.
